## 三、个人职责

### 3.1 需求分析与产品设计

需求调研：调研大学生心理健康需求，确定产品方向和核心功能

产品设计：设计 App 功能架构、页面交互流程和 UI 风格

版本规划：制定版本迭代计划，分阶段完成功能开发

### 3.2 系统架构设计

架构设计：设计 C/S 架构，确定前后端技术选型

接口设计：设计 RESTful API 接口规范和数据格式

数据库设计：设计 SQLite 数据库表结构和索引策略

数据同步：设计前后端数据同步和离线存储方案

### 3.3 Android 客户端开发

原生开发：使用 Java 开发全部 Android 客户端代码

页面架构：开发 Fragment + Activity 页面架构和导航系统

交互开发：实现心情调音台、测评答题、匿名广场等核心交互

自定义 View：开发自定义 View 组件（折线图、柱状图、桌宠动画）

网络层：实现 OkHttp 网络请求封装和 JSON 数据解析

本地存储：使用 Room 数据库实现本地数据持久化

### 3.4 Flask 后端开发

API 开发：开发全部 RESTful API 接口（20+ 个接口）

安全机制：实现用户认证、密码加密、API 限流等安全机制

数据库：设计并实现 SQLite 数据库初始化和数据操作

社交功能：实现匿名广场、帖子排序、点赞等社交功能

数据同步：实现测评结果云端同步和多设备数据共享

### 3.5 测试与部署

接口测试：使用 curl 工具测试所有 API 接口

真机测试：在真机上测试 App 功能和兼容性

编译打包：使用 Gradle 编译生成 Release APK

服务器部署：将后端部署到阿里云 ECS 服务器并配置域名

自动化部署：使用 paramiko 实现自动化部署脚本

## 四、技术亮点

### 4.1 自定义 View 数据可视化

为实现心情趋势和测评结果的可视化，独立开发了多个自定义 View 组件：

MoodTrendView：使用 Canvas + Path 绘制折线图，支持贝塞尔曲线平滑

BarChartView：自定义柱状图组件，支持渐变色柱体和动画效果

交互设计：图表支持触摸交互，点击数据点显示详情

PetView：使用 Canvas 绘制桌宠动画，实现帧动画效果

### 4.2 匿名社交系统设计

设计并实现完整的匿名社交系统：

匿名机制：基于 device_id hash 生成固定匿名名，保护用户隐私

图文混排：支持图文混排发帖，图片压缩后上传

热度算法：服务端 SQL 实现热度排序（点赞数+时间加权）

点赞去重：点赞系统使用设备ID去重，防止刷赞

### 4.3 心理测评系统

实现专业的心理测评系统：

专业量表：内置 PHQ-9、GAD-7、PSS、睡眠质量 4 份国际通用量表

自动评估：根据得分自动计算等级和生成专业建议

综合报告：综合报告整合 4 份量表结果，给出整体评估

云端同步：测评结果自动同步到云端，支持多设备查看

### 4.4 番茄钟数据持久化

实现番茄钟数据的跨天持久化和统计：

日期归档：使用 SharedPreferences 按日期归档数据

统计计算：自动计算今日和本周的番茄数和专注时长

自动归档：跨天时自动将昨日数据归档到历史

宠物联动：每个番茄联动桌宠系统，增加 25 点经验

### 4.5 服务器自动化部署

实现后端代码的自动化部署：

SSH 通信：使用 Python paramiko 库实现 SSH 连接和文件传输

一键部署：编写自动化部署脚本，一键更新服务器代码

服务重启：自动重启 Gunicorn 服务，无需手动操作

自动验证：部署后自动验证接口可用性

### 4.6 安全机制实现

实现多层次的安全防护：

密码加密：使用 bcrypt 算法对密码进行哈希加密（加盐哈希）

API 限流：使用 Flask-Limiter 实现 API 请求频率限制

SQL 注入防护：所有 API 使用参数化查询，防止 SQL 注入

隐私保护：匿名广场隐藏真实用户名，保护用户隐私

### 4.7 跨平台数据同步

实现数据的云端同步和多设备共享：

云端同步：测评结果自动同步到云端服务器

去重机制：使用 UNIQUE 约束防止重复同步

多设备：换手机登录同一账号即可查看历史数据

本地备份：笔记和课表支持本地 JSON 导出备份

### 4.8 桌宠养成系统

设计并实现趣味桌宠养成系统：

多行为联动：心情记录、打卡、番茄钟等行为获取经验值

升级系统：经验值满自动升级，解锁新外观

情绪联动：宠物根据用户心情做出不同反应

动画实现：使用自定义 View 绘制宠物动画
